## AI エージェントとは

AI エージェント（AI Agent）とは、人間が文章で出した指示を受け取り、**ファイルの作成・編集・コマンドの実行などを自動的に行う AI ツール**です。指示は VSCode のチャット欄に日本語で入力するだけです。この「AI への指示文」のことを**プロンプト**と呼びます（このガイドでは「プロンプト」と書いてあったら「VSCode のチャット欄に入力する文章」のことだと思ってください）。

普段使うチャット AI（ChatGPT など）は「質問に答える」ことが中心ですが、AI エージェントは **あなたの PC 上で実際にファイルを作成したり、コードを書いたりできます**。つまり、「こういう MOD を作って」と指示するだけで、AI がフォルダを作り、スクリプトを書き、ファイルを正しい場所に配置してくれます。

### なぜ MOD 開発に AI エージェントが使えるのか

このリポジトリには `CLAUDE.md` という特別なファイルが含まれています。このファイルには、MOD のフォルダ構成・画像サイズ・Lua コードの書き方・用語の統一ルールなど、正しい MOD を作るために必要なルールがすべて書かれています。

AI エージェントは **VSCode でこのリポジトリを開くだけで `CLAUDE.md` を自動的に読み込み**、MOD 開発のルールを「学習」します。その結果、AI が生成するコードやフォルダ構成はガイドに準拠した正しいものになります。

> **なぜ CLAUDE.md が重要？**: `CLAUDE.md` は AI エージェントへの「教科書」です。MOD のファイル構成・画像サイズ・用語ルール・コードのテンプレートなど、AI が正しい MOD を作るために必要な情報がすべて書かれています。あなたがこのファイルを読む必要はありません。AI が自動的に読んで理解します。

> **補足**: `CLAUDE.md` というファイル名は Claude Code 専用の仕組みですが、ファイルの中身は MOD 開発ルールを記述した汎用的なドキュメントです。GitHub Copilot などの他の AI エージェントでも、`@workspace` 機能やプロンプト内で「CLAUDE.md のルールに従ってください」と指示すれば、同じルールを参照して正しい MOD を生成できます。

### このガイドで何ができるようになるか

- AI エージェントに指示を出して、キャラクター MOD のスクリプト一式を自動生成できる
- 人間が行う作業（画像作成・テスト・アップロード）と AI に任せる作業を明確に分けて、効率よく MOD を開発できる
- エラーが出たときに、ログを AI に渡すだけで原因特定と修正ができる

> **ポイント**: AI エージェントはあなたの代わりにコードを書いてくれますが、最終的な確認はあなた自身が行います。心配いりません。このガイドに沿って進めれば大丈夫です。

> **補足**: 手動で MOD を作りたい場合は [メインガイド](../README.md) を参照してください。このガイドで解説する内容は、メインガイドの手順を AI に代行させるものです。

## 人間が行う作業と AI が行う作業（全体像）

MOD 開発のすべてを AI だけで完結させることはできません。以下の表で、**人間が必ず行う作業**と**AI に任せられる作業**を把握してください。

### 作業分担の全体像

「優先度」の意味は以下の通りです。

- **必須**: MOD を動作させるために絶対に必要な作業
- **推奨**: なくても動くが、やると MOD の完成度が大きく上がる作業
- **任意**: お好みで追加できるオプション作業

| 作業 | 担当 | 優先度 | 理由 |
|---|---|---|---|
| リポジトリのダウンロード・展開 | 人間 | 必須 | ブラウザでの操作が必要 |
| VSCode / Node.js / Claude Code のインストール | 人間 | 必須 | ソフトウェアのインストールは手動で行う |
| Steam + DST + Don't Starve Mod Tools のインストール | 人間 | 必須 | ゲーム本体とツール群が必要 |
| キャラクターの設定を考える（名前・性格・能力） | 人間 | 必須 | 創造的な判断が必要 |
| キャラクター画像の作成（ポートレート等） | 人間 | 必須 | AI はイラストを描けない。仮画像でもテストは可能だが、最終的には必要 |
| アニメーション用スプライトシートの作成 | 人間 | 推奨 | なくてもウィルソンの見た目で動作するが、オリジナルの外見にするには必要 |
| Spriter でのアニメーション確認・調整 | 人間 | 推奨 | GUI ツールの操作が必要。見た目の品質を上げるためにやった方が良い |
| MOD フォルダ構成の作成 | AI | 必須 | ガイドに従って自動生成 |
| modinfo.lua の作成 | AI | 必須 | テンプレートに基づき自動生成 |
| modmain.lua の作成 | AI | 必須 | ガイドに従って自動生成 |
| prefab ファイル（キャラクター定義）の作成 | AI | 必須 | キャラクター能力を Lua で実装 |
| Speech ファイル（セリフ）の作成 | AI | 推奨 | なくても動くが、セリフがないと味気ない MOD になる |
| PNG → TEX 変換コマンドの生成 | AI | 必須 | コマンドを生成（実行は人間が確認して実行） |
| ファイル配置 | AI | 必須 | 正しい場所にファイルを作成・移動 |
| ゲーム内テスト | 人間 | 必須 | DST を起動してプレイ確認 |
| エラーログの取得 | 人間 | 任意 | エラー発生時にログを AI に渡すと修正が早い |
| エラーログの解析・コード修正 | AI | 任意 | ログを渡せば原因特定と修正を実行 |
| modinfo コンフィグオプションの追加 | AI | 任意 | ゲーム内でステータス等を設定できるようにする |
| Workshop 用プレビュー画像の作成 | 人間 | 推奨 | 公開する場合は必要。公開しないなら不要 |
| Workshop への実際のアップロード | 人間 | 任意 | 公開したい場合のみ。Steam クライアントの操作が必要 |
| サウンド・ボイスの作成 | 人間 | 任意 | キャラクターに声を付けたい場合のみ。FMOD Designer での作業や音声録音 |

### AI だけでは作成できないもの

以下は AI エージェントの能力の限界であり、**必ず人間が行う必要があります**。

1. **画像・イラストの作成**: AI エージェント（Claude Code / GitHub Copilot）はテキスト処理・コード生成の AI であり、**イラストを描く機能はありません**。キャラクターの立ち絵、ポートレート、アイコンなどの画像はすべて人間が用意する必要があります。
2. **Spriter でのアニメーション設定**: Spriter は GUI（グラフィカルユーザーインターフェース）ベースのツールであり、AI エージェントからは操作できません。
3. **ゲーム内テスト**: DST を実際に起動してキャラクターが正しく表示・動作するかの確認は人間が行います。
4. **Steam Workshop へのアップロード**: Mod Uploader ツールの操作は人間が行います。
5. **サウンドの作成**: 音声ファイルの録音や FMOD Designer での編集は人間が行います。

> **重要**: AI エージェントは「コードを書く」「ファイルを配置する」「エラーを解析する」ことは得意ですが、「絵を描く」「ゲームを起動する」「外部ツールの GUI を操作する」ことはできません。この点を理解した上で、AI に任せる部分と人間が行う部分を使い分けましょう。

### 事前に人間が準備するもの（チェックリスト）

MOD 作成を AI に依頼する前に、以下を準備してください。

- [ ] Steam + Don't Starve Together がインストール済み
- [ ] Don't Starve Mod Tools がインストール済み（Steam ライブラリの「ツール」から無料でインストール）
- [ ] VSCode（Visual Studio Code）がインストール済み
- [ ] Node.js（v18 以上）がインストール済み（Claude Code の動作に必要）
- [ ] Claude アカウント + Claude Code のサブスクリプション（または GitHub アカウント + GitHub Copilot）
- [ ] キャラクターの構想（名前・性格・ステータス・特殊能力のアイデア）
- [ ] キャラクター画像（最低限ポートレート画像。まずは仮画像でも OK）

> **ポイント**: 画像はあとから差し替えられます。まずは仮の画像（単色の正方形など）で AI に MOD を作らせてゲーム内テストし、動作を確認してから本番の画像を用意する方法がおすすめです。

## セットアップ手順

### Step 1: このリポジトリの入手

MOD 開発ガイドが詰まったこのリポジトリを、あなたの PC にダウンロードします。2 つの方法があります。

#### 方法 A: ZIP ダウンロード（Git 不要、初心者向け）

1. ブラウザで `https://github.com/pinpikokun/DST-Mod-Guide` を開く
2. 画面右上の緑色の **Code** ボタンをクリック
3. **Download ZIP** をクリック
4. ダウンロードした ZIP ファイルを右クリック
5. **すべて展開** を選択
6. 展開先フォルダを確認して **展開** をクリック

> **注意**: ZIP を展開すると `DST-Mod-Guide-main` というフォルダの中にさらに同名のフォルダが入っている場合があります（二重フォルダ）。中を開いて `CLAUDE.md` や `README.md` が直接見えるフォルダを使ってください。

#### 方法 B: git clone（Git がインストール済みの場合）

Git Bash やコマンドプロンプトで以下のコマンドを実行します。

```bash
git clone https://github.com/pinpikokun/DST-Mod-Guide.git
```

> **どちらを選べば良い？**: Git を使ったことがなければ、方法 A（ZIP ダウンロード）で問題ありません。どちらの方法でも、このガイドの内容は同じように使えます。

### Step 2: VSCode のインストール

VSCode（Visual Studio Code）は、AI エージェントを動かすために必要なコードエディタです。

既にインストール済みの場合は、このステップをスキップしてください。

1. ブラウザで `https://code.visualstudio.com/` を開く
2. **Download for Windows** ボタンをクリック
3. ダウンロードしたインストーラーを実行する
4. 使用許諾に同意し、デフォルト設定のまま **次へ** を押し続ける
5. **インストール** をクリック
6. インストールが完了したら **完了** をクリック

> **なぜ VSCode？**: VSCode は無料のコードエディタで、Claude Code や GitHub Copilot などの AI ツールの拡張機能が充実しています。メモ帳やサクラエディタでもコードは書けますが、AI エージェントを使うには VSCode が必要です。

### Step 3: Claude Code のインストール

Claude Code を使うには、Node.js と Claude Code 拡張機能の 2 つをインストールします。

#### 3a. Node.js のインストール

Node.js は Claude Code を動かすために必要なソフトウェアです。

1. ブラウザで `https://nodejs.org/` を開く
2. **LTS**（Long Term Support、長期サポート版）のダウンロードボタンをクリック
3. ダウンロードしたインストーラーを実行する
4. デフォルト設定のまま **Next** を押し続ける
5. **Install** をクリック
6. インストールが完了したら **Finish** をクリック

インストール後、正しくインストールされたか確認します。

1. VSCode を開く
2. 画面上部メニューの **ターミナル** → **新しいターミナル** をクリック
3. 開いたターミナルに以下を入力して Enter キーを押す

```bash
node --version
```

`v18.0.0` 以上のバージョン番号が表示されれば OK です（例: `v24.13.1`）。

> **なぜ Node.js が必要？**: Claude Code は内部的に Node.js を使って動作しています。Node.js 自体を直接操作することはないので、インストールするだけで大丈夫です。

#### 3b. Claude Code VSCode 拡張機能のインストール

1. VSCode を開く
2. 左側のサイドバーにある **拡張機能** アイコン（四角が 4 つ並んだマーク）をクリック
3. 上部の検索バーに `Claude Code` と入力する
4. **Anthropic** が提供する「Claude Code」を見つける
5. **インストール** ボタンをクリック
6. インストールが完了すると、左のサイドバーに Claude のアイコンが表示される

> **補足**: 初回起動時に Claude アカウントへのログインを求められます。画面の指示に従ってブラウザでログインしてください。ログインが完了すると VSCode に自動的に戻ります。

#### 代替ツール: GitHub Copilot（無料プランあり）

Claude Code の代わりに GitHub Copilot を使うこともできます。

1. VSCode の拡張機能で `GitHub Copilot` を検索してインストール
2. GitHub アカウントでログイン
3. Copilot Chat パネルを開き、**Agent モード**（`@workspace` を使った指示）で MOD 作成を依頼する

> **ポイント**: このガイドでは主に Claude Code を使って説明しますが、GitHub Copilot の Agent モードでも同様の手順で MOD を作成できます。Copilot で操作する場合は、始める前にチャット欄に `@workspace` と打ってください。CLAUDE.md を読み込ませて AI エージェントにルールを理解させることができます。

### Step 4: AI エージェントにドキュメントを読み込ませる

ここが最も重要なステップです。AI エージェントに MOD 開発ガイドを読み込ませます。

1. VSCode を開く
2. 画面上部メニューの **ファイル** → **フォルダーを開く** をクリック
3. Step 1 で展開（またはクローン）した **DST-Mod-Guide** フォルダを選択する
4. **フォルダーの選択** をクリック
5. 左のサイドバーにリポジトリのファイル一覧が表示されることを確認する

> **重要**: 必ず `CLAUDE.md` が見えるフォルダ（リポジトリのルートフォルダ）を開いてください。サブフォルダ（例: `DST-Spriter/` だけ）を開いても AI は正しく動作しません。

これで準備完了です。Claude Code のサイドパネルを開けば、AI エージェントに指示を出せます。

> **なぜフォルダを開くだけで良いのか？**: Claude Code は VSCode で開いたフォルダ内の `CLAUDE.md` を自動的に読み込みます。このファイルに MOD 開発の全ルールが記載されているため、AI は「DST MOD の正しい作り方」を理解した状態で応答してくれます。

## MOD 作成の指示方法

ここからが本番です。AI エージェントに指示を出して、実際に MOD を作成していきます。

各ステップで **AI に任せる部分** と **人間が行う部分** を明確にしています。`> **[人間の作業]**:` と書かれた部分は、あなた自身が手を動かす必要があるパートです。

### Step 5: キャラクター MOD の基本情報を伝える

まず、作りたいキャラクターの情報を AI に伝えます。以下の情報を考えておいてください。

- **キャラクター名（英語、小文字）**: ゲーム内部で使う名前（例: `sakura`）
- **表示名**: ゲーム内で表示される名前（例: サクラ）
- **説明文**: キャラクター選択画面に表示される説明
- **体力 / 空腹 / 正気度**: 各ステータスの初期値
- **特殊能力**（あれば）: キャラクター固有の能力

> **[人間の作業]**: キャラクターの設定を考えるのは人間の仕事です。AI は設定を「実装」しますが、「どんなキャラクターにするか」はあなたが決めます。

Claude Code のチャットパネルに、以下のようなプロンプトを入力してください。

```
以下の設定でキャラクター MOD を作成してください。
このリポジトリのガイド（README.md および各サブフォルダのガイド）に従って、
MOD のフォルダ構成とファイルを一式作成してください。

- キャラクター名（内部名）: sakura
- 表示名: サクラ
- 説明文: 桜の精霊。春が来ると力を増す。
- 体力: 200
- 空腹: 150
- 正気度: 120
- 特殊能力: 春の間、移動速度が 1.2 倍になる

MOD フォルダは以下の場所に作成してください:
C:\Program Files (x86)\Steam\steamapps\common\Don't Starve Together\mods\sakura
```

> **ポイント**: 最初は特殊能力なしの「見た目だけ変更したキャラクター」から始めることをおすすめします。動作確認ができてから能力を追加するほうが、エラーの原因を特定しやすくなります。

### Step 6: AI に MOD の雛形を作成させる

Step 5 のプロンプトを送ると、AI は以下のファイルを自動生成します。

- `modinfo.lua` — MOD の基本情報
- `modmain.lua` — MOD のエントリーポイント
- `scripts/prefabs/<キャラクター名>.lua` — キャラクター定義
- `scripts/speech_<キャラクター名>.lua` — キャラクターのセリフ

### Step 7: 画像の準備と配置

このステップは **人間の作業が最も多い** パートです。

> **[人間の作業]**: キャラクター画像の作成は AI にはできません。ペイント、GIMP、Krita、Photoshop などの画像編集ソフトを使って、以下の画像を用意してください。

#### 必要な画像一覧

| 画像の用途 | サイズ | 保存先（MOD フォルダ内） |
|---|---|---|
| キャラクター選択画面（大） | 256 x 512 | `images/selectscreen_portraits/` |
| キャラクター選択画面（セーブスロット） | 128 x 128 | `images/saveslot_portraits/` |
| キャラクター詳細画面 | 512 x 512 以上 | `images/bigportraits/` |
| マップアイコン | 64 x 64 | `images/map_icons/` |
| アバター | 64 x 64 | `images/avatars/` |
| MOD アイコン | 任意（推奨 256 x 256） | MOD ルート直下 |
| アニメーション用スプライトシート | キャラクターによる | `exported/` フォルダ |

> **重要**: すべての画像は**各辺が 2 の累乗**（64, 128, 256, 512 など）である必要があります。例: 256 x 512 は OK ですが、300 x 400 は NG です。

> **初心者向けTip:** まだ画像を用意していなくても大丈夫です。AI に「仮画像でテストする方法を教えてください」と聞けば、単色の仮画像を使ったテスト方法を案内してくれます。本番の画像はあとから差し替えられます。

#### 顔だけ差し替えたい場合のヒント

「全部のパーツを描くのは大変だけど、せめて顔だけはオリジナルにしたい」という場合は、ウィルソンのパーツ画像をテンプレートとして使えます。

DST Mod Tools には、ウィルソンの身体パーツが分かれた PNG 画像が同梱されています。以下のフォルダに入っています。

```
C:\Program Files (x86)\Steam\steamapps\common\Don't Starve Mod Tools\mod_tools\sample_build\
```

この中にある `face/` フォルダにウィルソンの顔の PNG 画像（`face-0.png` 〜 `face-13.png`）が入っています。これらの画像をペイントソフトで開き、**輪郭や目・口を自分の好きな顔に描き替える** だけで、オリジナルの顔を持つキャラクターが作れます。

> **ポイント**: 顔以外のパーツ（体・腕・脚など）はウィルソンのまま残しても問題ありません。まずは顔だけ差し替えて動作を確認し、あとから他のパーツも少しずつ描き替えていくのがおすすめです。

> **補足**: `sample_build` フォルダの中身や Spriter での使い方は [メインガイドの Step 6](../README.md#step-6-キャラクター画像を用意するsample_build) で詳しく解説しています。

画像の準備ができたら、AI に配置を依頼します。

```
以下の画像ファイルを用意しました。
MOD フォルダ内の正しい場所に配置してください。
また、PNG から TEX への変換が必要なファイルについて、変換コマンドを教えてください。

- selectscreen_sakura.png（256 x 512）
- saveslot_sakura.png（128 x 128）
- bigportrait_sakura.png（512 x 512）
- map_icon_sakura.png（64 x 64）
- avatar_sakura.png（64 x 64）
- modicon.png（256 x 256）
```

> **[人間の作業]**: PNG から TEX への変換コマンドは AI が生成しますが、コマンドの実行自体は人間が確認してから行ってください。AI がターミナルでの実行を提案した場合は、内容を確認した上で許可してください。

画像変換の詳細は [DST-TextureConverter ガイド](../DST-TextureConverter/README.md) を参照してください。

### Step 8: AI にスクリプトを書かせる

キャラクターの能力やセリフの追加・修正を AI に依頼します。

#### prefab ファイル（キャラクター能力）

> **なぜ prefab（プレハブ）？**: DST では、キャラクター・アイテム・建造物など、ゲーム内に登場するすべてのモノを「prefab（プレハブ）」と呼びます。「あらかじめ組み立てられた部品」という意味で、キャラクターの体力・空腹度・特殊能力などをひとまとめに定義したファイルが prefab ファイルです。

```
prefab ファイル（scripts/prefabs/sakura.lua）に以下の能力を実装してください。
DST-Lua-Scripting ガイドに従ってください。

- 春の間、移動速度が 1.2 倍になる
- 季節が変わったら速度が元に戻る
- 季節の変化はイベントリスナーで検知する
```

#### Speech ファイル（セリフ）

```
Speech ファイル（scripts/speech_sakura.lua）を作成してください。
DST-Lua-Scripting/README-speech.md に従った形式で、
キャラクターの性格（桜の精霊、穏やかで詩的な話し方）に合ったセリフを書いてください。
全カテゴリのセリフを網羅してください。
```

#### modinfo のコンフィグオプション

> **なぜコンフィグオプション？**: コンフィグオプションを追加すると、MOD 選択画面でボルトマーク（設定アイコン）を押したときに、キャラクターの体力や能力の強さなどをプレイヤーが自由にカスタマイズできるようになります。たとえば「体力を 100 / 150 / 200 から選べる」といった設定を、コードを書き換えずにゲーム内の画面から変更できます。必須ではありませんが、追加しておくと遊ぶ人にとても喜ばれます。

```
modinfo.lua にコンフィグオプションを追加してください。
DST-Lua-Scripting/README-modinfo-config.md に従ってください。

追加するオプション:
- 体力の初期値（100 / 150 / 200 / 250 から選択、デフォルト: 200）
- 春の移動速度ボーナス（1.0 / 1.2 / 1.5 から選択、デフォルト: 1.2）
```

> **ポイント**: AI に依頼するときは、参照すべきガイドの名前を具体的に伝えると精度が上がります。「DST-Lua-Scripting/README-speech.md に従って」のように指定してください。

### Step 10: テストとデバッグ

MOD が完成したら、ゲーム内でテストします。

> **[人間の作業]**: 以下のテスト作業はすべて人間が行います。

1. DST を起動する
2. メインメニューで **MOD** をクリック
3. 作成した MOD を見つけて **有効化** する
4. 新しいワールドを作成してゲームを開始する
5. キャラクター選択画面で作成したキャラクターを選択する
6. ゲーム内でキャラクターが正しく表示されるか確認する
7. 特殊能力が動作するか確認する

#### エラーが出た場合

ゲームがクラッシュしたり、キャラクターが正しく表示されない場合は、以下の 2 つの方法で AI にエラーを解析させることができます。

##### 方法 1: エラー画面のスクリーンショットを貼り付ける

DST でエラーが発生すると、ゲーム画面にエラーログが表示されて停止します。この画面のスクリーンショットを撮って、AI のチャット画面に貼り付けるだけで原因を調査できます。

1. エラー画面が表示されたら、`PrintScreen` キーまたは `Win + Shift + S` でスクリーンショットを撮る
2. AI のチャット画面に画像を貼り付ける（VSCode の Claude Code チャットにドラッグ＆ドロップ、またはペースト）
3. 以下のプロンプトを送る

```
MOD を有効にしてゲームを起動したところ、エラーが出ました。
添付のスクリーンショットを見て、原因と修正方法を教えてください。修正も実行してください。
```

##### 方法 2: AI にログファイルを直接読ませる

AI にログファイルの場所を教えて、直接ファイルを読んで解析してもらう方法です。スクリーンショットが撮れない場合や、より詳細な情報が必要な場合に便利です。

```
MOD を有効にしてゲームを起動したところ、エラーが出ました。
以下のログファイルを読んで、原因を特定して修正してください。

C:\Users\<ユーザー名>\Documents\Klei\DoNotStarveTogether\client_log.txt
```

> **ヒント**: `<ユーザー名>` の部分は、ご自身の Windows ユーザー名に置き換えてください。エクスプローラーのアドレスバーに `%USERPROFILE%\Documents\Klei\DoNotStarveTogether` と入力すると直接開けます。

AI がエラーの原因を特定し、コードを自動修正してくれます。修正が完了したら、再度ゲームを起動してテストしてください。

> **ポイント**: 方法 1（スクリーンショット）が最も手軽です。画像を貼り付けるだけで AI がエラー内容を読み取って修正してくれます。

### Step 11: Workshop 公開の準備

MOD の動作確認ができたら、Workshop への公開準備を AI に依頼します。

```
MOD を Steam Workshop に公開する準備をしてください。
以下を確認・実行してください:

1. modinfo.lua の公開用設定が正しいか確認
2. 不要なファイル（exported/ フォルダ等）の一覧を教えて
3. Workshop 用の説明文（英語）を作成して
```

> **[人間の作業]**: 以下は人間が行う作業です。
> - Workshop 用プレビュー画像（JPG または PNG、512 x 512 以上推奨）の作成
> - Mod Uploader ツールの起動と操作

Workshop 公開の詳しい手順は [DST-Workshop-Upload ガイド](../DST-Workshop-Upload/README.md) を参照してください。

## よくあるトラブルとエラー対処法

| 症状 | 原因 | 解決法 |
|---|---|---|
| Claude Code が応答しない | API 接続の問題またはサブスクリプションの問題 | Claude のダッシュボードでアカウント状態を確認する |
| AI が間違ったフォルダ構成を作る | `CLAUDE.md` が読み込まれていない | VSCode でリポジトリのルートフォルダ（`CLAUDE.md` が見えるフォルダ）を開き直す |
| AI が生成したコードでゲームがクラッシュする | Lua の文法エラーまたは API の使い方が間違っている | エラーログを AI に渡して修正を依頼する |
| 「CLAUDE.md が見つかりません」と言われる | フォルダの開き方が間違っている | ZIP を展開した場合、二重フォルダになっていないか確認する（`CLAUDE.md` が直接見えるフォルダを選ぶ） |
| AI が英語で返答する | 日本語で指示していない | プロンプトの冒頭に「日本語で回答してください」と追加する |
| Node.js のバージョンエラー | Claude Code が v18 以上を要求 | Node.js の公式サイトから最新の LTS 版をインストールし直す |
| GitHub Copilot がコード補完しかしない | Agent モードが有効になっていない | Copilot Chat パネルを開き、`@workspace` を使って指示する |
| TEX 変換がうまくいかない | TextureConverter のパスが間違っている | パスをコピーして AI に確認してもらう |
| 画像が 2 の累乗でないというエラー | 画像サイズが 64, 128, 256, 512 のいずれかでない | 画像を正しいサイズにリサイズする |

## Tips・参考情報

### Claude Code・GitHub Copilot・Google Antigravity の比較

| 項目 | Claude Code | GitHub Copilot | Google Antigravity |
|---|---|---|---|
| 料金 | 有料（MAX プラン） | 無料プラン（月間制限あり）/ 有料プラン | 無料（パブリックプレビュー） |
| 実行環境 | VSCode 拡張機能 | VSCode 拡張機能 | 専用 IDE（VSCode ベース） |
| ファイル操作 | 直接ファイルを作成・編集可能 | Agent モードで可能 | エージェントが自律的に作成・編集 |
| コンテキスト理解 | `CLAUDE.md` を自動読み込み | `@workspace` でプロジェクト参照 | エージェントがプロジェクト全体を参照 |
| 公式サイト | [claude.ai](https://claude.ai/) | [GitHub Copilot](https://github.com/features/copilot?locale=ja) | [Google Antigravity](https://antigravity.google/) |

### AI エージェントと効果的にやり取りするコツ

1. **一度に全部を頼まず、ステップごとに進める**: 「MOD を全部作って」よりも「まず modinfo.lua を作って」→「次に prefab を作って」と段階的に進めるほうが、エラーが少なくなります。
2. **参照すべきガイドを指定する**: 「DST-Lua-Scripting/README-speech.md に従って」のように、具体的なガイド名を指定すると精度が上がります。
3. **エラーログは省略しない**: エラーが出たら、ログ全文を貼り付けてください。「エラーが出ました」だけでは AI は対応できません。
4. **AI の出力を必ず確認する**: AI が生成したコードは高品質ですが、100% 正しいとは限りません。必ずゲーム内でテストしてください。
5. **うまくいかない場合は言い方を変える**: 同じプロンプトで何度やってもうまくいかない場合は、指示の仕方を変えてみてください。具体的に「何が問題で、どうしてほしいか」を伝えると改善します。

### 情報の鮮度について
