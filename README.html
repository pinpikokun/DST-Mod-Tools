Character MOD Guide

DST MOD 開発ガイド
はじめてのキャラクター MOD を作ろう

Don't Starve Together のキャラクター MOD を作りたい人のための初心者向け総合ガイド。
ウィルソンをベースに、コピペ＋名前の書き換えだけでオリジナルキャラクターを作れます。

About

このガイドについて

このガイドでは、ウィルソン（ゲーム内の基本キャラクター）をベースに見た目だけを変更したオリジナルキャラクターを作成します。プログラミング経験は不要です。

対象 OS Windows 11
プログラミング 不要
作業方法 コピペ＋書換
前提 DST プレイ経験

ゲーム内アニメーション

Installation

必要なツールのインストール

1. Don't Starve Mod Tools（必須・無料）

MOD 開発に必要なツールがすべて入ったパッケージです。

  1. Steam を開く
  2. 上部メニューの「ライブラリ」をクリック
  3. 検索バーに Don't Starve Mod Tools と入力（見つからない場合はフィルターアイコンから「ツール」にチェック）
  4. 「Don't Starve Mod Tools」をクリックし「インストール」ボタンを押す
インストール先

C:\Program Files (x86)\Steam\steamapps\common\Don't Starve Mod Tools\mod_tools\

含まれる主なツール

ツール場所用途
Spritermod_tools\Spriter\アニメーション作成・編集
FMOD Designermod_tools\FMOD_Designer\サウンド/ボイス作成
TextureConvertermod_tools\tools\bin\TextureConverter.exePNG → TEX 画像変換
autocompilermod_tools\autocompiler.exeアニメーション自動ビルド
sample_buildmod_tools\sample_build\ウィルソンのパーツ画像テンプレート
2. Git for Windows（Git Bash）

TextureConverter を使うために Git Bash が必要です。Windows 標準のコマンドプロンプトでも動きますが、パスにスペースが含まれる場合の扱いが面倒なため Git Bash を推奨します。

  1. https://gitforwindows.org/ にアクセスし「Download」をクリック
  2. ダウンロードした Git-*.*.*-64-bit.exe を実行（設定はすべてデフォルトで OK）
  3. インストール後、フォルダの空いている場所で右クリック →「Open Git Bash here」が表示されれば成功
Windows 11 の場合

右クリックメニューに直接表示されないことがあります。「その他のオプションを確認」をクリックすると表示されます。

3. テキストエディタ（VSCode 推奨）

Lua コードを編集するために必要です。メモ帳でも可能ですが、色分け表示やエラー検出ができる VSCode を強く推奨します。

  1. https://code.visualstudio.com/ にアクセス
  2. 「Download for Windows」をクリック
  3. ダウンロードしたインストーラーを実行（デフォルト設定のまま）
Folder Structure

MOD フォルダ構成の全体像

キャラクター MOD は決まったフォルダ構成を持っています。my_character の部分はすべて自分のキャラクター名に統一してください。

my_character/ ← MOD フォルダ（この名前は自由）
├── modinfo.lua ← MOD の情報（名前・説明・バージョン）
├── modmain.lua ← MOD のメインスクリプト
├── modicon.tex / .xml ← MOD 一覧に表示されるアイコン
│
├── anim/ ← アニメーション ZIP ファイル
│   ├── my_character.zip
│   └── ghost_my_character_build.zip
│
├── bigportraits/ ← 大きなポートレート画像
│   └── my_character.tex / .xml
│
├── images/ ← 各種画像ファイル
│   ├── avatars/ ← チャットアバター
│   │   ├── avatar_my_character.tex / .xml
│   │   └── avatar_ghost_my_character.tex / .xml
│   ├── map_icons/ ← ミニマップアイコン
│   │   └── my_character.tex / .xml
│   ├── saveslot_portraits/ ← セーブスロットのポートレート
│   │   └── my_character.tex / .xml
│   └── selectscreen_portraits/ ← キャラ選択画面のポートレート
│       └── my_character.tex / .xml
│
├── scripts/ ← Lua スクリプト
│   ├── prefabs/
│   │   └── my_character.lua ← キャラクターの定義
│   └── speech_my_character.lua ← セリフ
│
└── sound/ ← サウンド（オプション）
重要

すべて小文字で統一してください。大文字・小文字の不一致はクラッシュの原因になります。

MOD の読み込み場所

DST はゲーム本体の mods フォルダから MOD を読み込みます。

  1. Steam を開く
  2. ライブラリから Don't Starve Together を右クリック
  3. 管理 → ローカルファイルを閲覧 をクリック
  4. 開いたフォルダ内の mods フォルダが MOD の配置先
パス

C:\Program Files (x86)\Steam\steamapps\common\Don't Starve Together\mods\

Steam のインストール先を変更している場合はパスが異なります。上記の手順で Steam から開けば正しい場所が開きます。

Step 1

MOD フォルダを作成する

この章からキャラクター名を my_character として進めます。自分のキャラクター名に置き換えてください。

命名ルール

半角英数字とアンダースコア _ のみ使用可能。すべて小文字を推奨。

スペース・日本語・ハイフン - は使用不可。

例: my_character、fire_knight、sakura_witch

  1. 以下のフォルダを開く: C:\Program Files (x86)\Steam\steamapps\common\Don't Starve Mod Tools\mod_tools\
  2. このフォルダ内に my_character という名前のフォルダを作成
  3. my_character フォルダの中にサブフォルダを作成
my_character/
├── anim/
├── bigportraits/
├── images/
│   ├── avatars/
│   ├── map_icons/
│   ├── saveslot_portraits/
│   └── selectscreen_portraits/
├── scripts/
│   └── prefabs/
└── sound/
なぜここに作るの？

この場所に MOD フォルダを置くと、autocompiler.exe がアニメーションを自動的にビルドしてくれます。開発が完了したら、ゲームの mods フォルダにコピーします。

Step 2

modinfo.lua を作成する

MOD の基本情報を定義するファイルです。以下のコードをコピペして、★マークの部分を自分のキャラクター名に書き換えてください。

-- MOD の基本情報
name = "My Character"                    -- ★書き換え: MOD の名前
description = "My first DST character!"  -- ★書き換え: MOD の説明文
author = "Your Name"                     -- ★書き換え: 作者名
version = "1.0.0"

-- DST の API バージョン（そのまま）
api_version = 10

-- DST 専用 MOD（そのまま）
dont_starve_compatible = false
reign_of_giants_compatible = false
dst_compatible = true

-- MOD アイコン（そのまま）
icon_atlas = "modicon.xml"
icon = "modicon.tex"

-- マルチプレイ設定（そのまま）
all_clients_require_mod = true
clients_only_mod = false

-- 設定オプション（そのまま）
configuration_options = {}
各行の意味

name: Steam Workshop やゲーム内 MOD 一覧に表示される名前

api_version = 10: DST の現在の API バージョン（変更不要）

all_clients_require_mod = true: マルチプレイで全員がこの MOD を必要とする設定

Step 3

modmain.lua を作成する

MOD のメインスクリプトです。★マークのある my_character をすべて自分のキャラクター名に書き換えてください。

modmain.lua の全コード
-- ========================================
-- アセット（画像・アニメーション）の登録
-- ★以下すべての "my_character" を自分のキャラクター名に書き換えてください
-- ========================================

Assets = {
    -- キャラクターアニメーション
    Asset("ANIM", "anim/my_character.zip"),                   -- ★書き換え
    Asset("ANIM", "anim/ghost_my_character_build.zip"),           -- ★書き換え

    -- キャラクター選択画面の画像
    Asset("IMAGE", "images/saveslot_portraits/my_character.tex"),    -- ★書き換え
    Asset("ATLAS", "images/saveslot_portraits/my_character.xml"),    -- ★書き換え
    Asset("IMAGE", "images/selectscreen_portraits/my_character.tex"), -- ★書き換え
    Asset("ATLAS", "images/selectscreen_portraits/my_character.xml"), -- ★書き換え
    Asset("IMAGE", "bigportraits/my_character.tex"),                -- ★書き換え
    Asset("ATLAS", "bigportraits/my_character.xml"),                -- ★書き換え

    -- アバター
    Asset("IMAGE", "images/avatars/avatar_my_character.tex"),       -- ★書き換え
    Asset("ATLAS", "images/avatars/avatar_my_character.xml"),       -- ★書き換え
    Asset("IMAGE", "images/avatars/avatar_ghost_my_character.tex"), -- ★書き換え
    Asset("ATLAS", "images/avatars/avatar_ghost_my_character.xml"), -- ★書き換え

    -- マップアイコン
    Asset("IMAGE", "images/map_icons/my_character.tex"),            -- ★書き換え
    Asset("ATLAS", "images/map_icons/my_character.xml"),            -- ★書き換え
}

-- ========================================
-- キャラクター情報の設定
-- ========================================
local STRINGS = GLOBAL.STRINGS

STRINGS.CHARACTER_TITLES.my_character = "The Custom Character"           -- ★書き換え（称号）
STRINGS.CHARACTER_NAMES.my_character = "My Character"                    -- ★書き換え（表示名）
STRINGS.CHARACTER_DESCRIPTIONS.my_character = "*Has custom appearance\n*Based on Wilson"  -- ★書き換え
STRINGS.CHARACTER_QUOTES.my_character = "\"Hello, world!\""              -- ★書き換え
STRINGS.CHARACTER_ABOUTME.my_character = "A custom character."           -- ★書き換え

-- セリフデータの読み込み
STRINGS.CHARACTERS.MY_CHARACTER = require "speech_my_character"          -- ★大文字・小文字の両方を書き換え

-- キャラクター名
STRINGS.NAMES.MY_CHARACTER = "My Character"                              -- ★大文字で書き換え

-- スキン名
STRINGS.SKIN_NAMES.my_character_none = "My Character"                    -- ★書き換え

-- ========================================
-- キャラクターを登録する
-- ========================================
AddModCharacter("my_character", "MALE")  -- ★書き換え（"MALE" or "FEMALE"）
重要: 大文字・小文字の使い分け

小文字（my_character）: ファイル名、prefab 名、フォルダ名

大文字（MY_CHARACTER）: STRINGS テーブルのキー、キャラクター内部定数

この対応を間違えるとゲームがクラッシュします。

Step 4

キャラクター prefab ファイルを作成する

prefab（プレハブ）とは？

DST の世界に存在するすべてのもの（キャラクター、アイテム、モンスター、木、石など）は「prefab」として定義されています。prefab は「設計図」のようなもので、見た目・能力・ステータスをひとまとめにした定義です。

scripts/prefabs/my_character.lua として保存してください（ファイル名も書き換え）。

prefab ファイルの全コード
local MakePlayerCharacter = require("prefabs/player_common")

local assets = {
    Asset("ANIM", "anim/my_character.zip"),             -- ★書き換え
    Asset("ANIM", "anim/ghost_my_character_build.zip"), -- ★書き換え
}

-- 初期所持アイテム（空 = デフォルト）
local start_inv = {}  -- ★変更任意: 例 {"flint", "twigs"}

-- サーバー・クライアント両方で実行される処理
local common_postinit = function(inst)
    inst.MiniMapEntity:SetIcon("my_character.tex")  -- ★書き換え
end

-- サーバーのみで実行される処理
local master_postinit = function(inst)
    inst.components.health:SetMaxHealth(150)   -- ★変更任意: 体力
    inst.components.hunger:SetMax(150)         -- ★変更任意: 空腹
    inst.components.sanity:SetMax(200)         -- ★変更任意: 正気度

    inst.components.locomotor.walkspeed = 4   -- ★変更任意: 歩行速度
    inst.components.locomotor.runspeed = 6    -- ★変更任意: 走行速度

    inst.components.combat.damagemultiplier = 1  -- ★変更任意: 攻撃力倍率

    inst.soundsname = "wilson"  -- ★変更任意: "willow","wendy" など
end

return MakePlayerCharacter("my_character", prefabs, assets, common_postinit, master_postinit, start_inv)  -- ★書き換え
カスタマイズのヒント

SetMaxHealth(150) の数値を変えると体力が変わります（ウィルソン = 150、ウィグフリッド = 200）。

soundsname = "wilson" を "willow" や "wendy" に変えると、既存キャラのボイスを使えます。

start_inv にアイテムを追加すると初期所持アイテムを設定できます（例: {"flint", "flint", "twigs"}）。

Step 5

セリフファイルを作成する

scripts/speech_my_character.lua として保存してください。

-- ウィルソンのセリフをベースにした最小セリフファイル
-- 詳細なセリフを追加したい場合は、ウィルソンの speech ファイルをコピーして
-- 各セリフを自分のキャラクターに合った内容に書き換えてください。
--
-- ウィルソンの speech ファイルの場所:
-- C:\Program Files (x86)\Steam\steamapps\common\Don't Starve Together\data\databundles\scripts.zip
-- ↑この ZIP を展開すると scripts\speech_wilson.lua が見つかります

return require("speech_wilson")
解説

require("speech_wilson") はウィルソンのセリフデータをそのまま使うという意味です。オリジナルのセリフに変更したい場合は、ウィルソンの speech ファイルの内容をコピーして各行を書き換えてください。

Step 6

キャラクター画像を用意する

ウィルソンのパーツ画像を自分のキャラクターに差し替えます。

sample_build とは

DST Mod Tools には、ウィルソンの身体パーツが分かれた PNG 画像のテンプレートが含まれています。

場所

C:\Program Files (x86)\Steam\steamapps\common\Don't Starve Mod Tools\mod_tools\sample_build\

sample_build のフォルダ一覧
フォルダ中身説明
arm_lower/arm_lower-0.png ~ 2.png前腕（3パターン）
arm_upper/arm_upper-0.png ~ 4.png上腕（5パターン）
cheeks/cheeks-0.png ~ 1.png頬（食べるアニメーション用）
face/face-0.png ~ 13.png顔の表情（14パターン）
foot/foot-0.png ~ 6.png足（7パターン）
hair/hair-0.png ~ 2.png髪（3パターン）
hair_hat/hair_hat-0.png ~ 2.png帽子を被ったときの髪
hand/hand-0.png ~ 10.png手（11パターン）
headbase/headbase-0.png ~ 2.png頭の輪郭（3パターン）
headbase_hat/headbase_hat-0.png ~ 2.png帽子を被ったときの頭
leg/leg-0.png ~ 8.png脚（9パターン）
torso/torso-0.png ~ 9.png胴体（10パターン）
番号の意味

同じパーツでも正面・横向き・後ろ向きなどで画像が異なります。例: face-0.png は正面の普通の表情、face-4.png は横向きの表情です。

画像を差し替える手順

  1. sample_build フォルダを丸ごとコピーして my_character_build という名前で配置（この名前が Build 名になります）
  2. コピーした build.scml を Spriter で開いて、パーツ画像の配置を確認
  3. フォルダ内の PNG を同じファイル名・同じサイズで自分の画像に差し替え
  4. 差し替えが終わったら build.scml を Spriter で開いて見た目を確認
Spriter の基本操作
  1. mod_tools\Spriter\Spriter.exe をダブルクリックして起動
  2. File → Open Project で my_character_build\build.scml を開く
  3. 画面中央にキャラクターが表示される
  4. 下部の ▶ ボタンでアニメーション再生、ドロップダウンでアニメーション切り替え
ヒント

このガイドでは Spriter はプレビュー確認のためだけに使います。画像の差し替えはエクスプローラーで PNG を上書きするだけで OK です。

差し替えのルール

ファイル名は絶対に変えない。画像サイズは元画像と同じにする。透過 PNG（RGBA 32bit）で保存する。

初心者向けアドバイス

最初は全パーツを差し替える必要はありません。まずは face/ フォルダの画像だけ差し替えて、顔だけ変えたキャラクターから始めるのがおすすめです。

Step 7

アニメーションをビルドする

差し替えた画像を DST が読み込める ZIP 形式に変換します。

方法 A: autocompiler を使う（推奨）
  1. my_character_build フォルダを mod_tools\exported\ の中に移動（exported フォルダが無い場合は作成）
  2. mod_tools\autocompiler.exe を実行（黒い画面が出る。処理完了まで待つ）
  3. ビルド成功で mod_tools\anim\my_character_build.zip が生成される
  4. この ZIP を MOD フォルダの anim/my_character.zip としてコピー
ファイル名の注意

ZIP のファイル名（拡張子を除いた部分）が Build 名になります。prefab で AnimState:SetBuild("my_character") と指定する場合、ZIP は my_character.zip である必要があります。

ゴーストアニメーション

キャラクターが死亡するとゴースト（幽霊）になります。最も簡単な方法は DST 既存のゴーストアニメーションを使うことです。

コピー元:
C:\Program Files (x86)\Steam\steamapps\common\Don't Starve Together\data\anim\ghost_wilson_build.zip

コピー先:
my_character\anim\ghost_my_character_build.zip
Step 8

ポートレート画像を TEX に変換する

DST では PNG 画像をそのまま使えません。TEX 形式に変換する必要があります。

必要な画像一覧

画像サイズ配置先フォルダ説明
modicon128 x 128MOD フォルダ直下MOD 一覧のアイコン
selectscreen portrait256 x 512images/selectscreen_portraits/キャラ選択画面の一覧
saveslot portrait128 x 128images/saveslot_portraits/セーブスロットのアイコン
bigportrait512 x 512 以上bigportraits/キャラ選択画面の大きな立ち絵
avatar64 x 64images/avatars/チャットアイコン
avatar (ghost)64 x 64images/avatars/ゴースト時のチャットアイコン
map icon64 x 64images/map_icons/ミニマップアイコン
画像サイズの重要ルール

すべての画像サイズは 2 の累乗（64, 128, 256, 512...）でなければなりません。それ以外のサイズはゲームがクラッシュします。

TEX への変換手順

  1. 作業用フォルダに PNG を置く
  2. フォルダ内で右クリック →「Open Git Bash here」
  3. 以下のコマンドで PNG を TEX に変換（ファイル名を変えて各画像で実行）
"/c/Program Files (x86)/Steam/steamapps/common/Don't Starve Mod Tools/mod_tools/tools/bin/TextureConverter.exe" \
  -i ファイル名.png -o ファイル名.tex -f bc3 -p opengl --mipmap --premultiply
コマンドオプションの解説
オプション意味
-i入力ファイル（PNG）
-o出力ファイル（TEX）
-f bc3圧縮形式（BC3 = DXT5、透過あり）
-p openglプラットフォーム
--mipmapミップマップ生成（必須）
--premultiplyアルファ値の事前乗算（必須）

XML アトラスファイルの作成

XML アトラスファイルとは？

TEX ファイル（画像データ）だけでは、ゲームはその画像をどう読み込めばいいかわかりません。XML アトラスファイルは「この TEX の中にどんな画像がどの位置にあるか」を記述した設定ファイルです。TEX と XML は常にセットで必要です。

合計 7 個の XML ファイルを作成します。

#ファイル名中に書く TEX ファイル名配置先フォルダ
1modicon.xmlmodicon.texMOD フォルダ直下
2my_character.xmlmy_character.texbigportraits/
3avatar_my_character.xmlavatar_my_character.teximages/avatars/
4avatar_ghost_my_character.xmlavatar_ghost_my_character.teximages/avatars/
5my_character.xmlmy_character.teximages/map_icons/
6my_character.xmlmy_character.teximages/saveslot_portraits/
7my_character.xmlmy_character.teximages/selectscreen_portraits/
XML ファイルの作り方（メモ帳を使用）
  1. Windows のメモ帳を開く（スタートメニューで「メモ帳」と検索）
  2. 以下のテンプレートをコピペ
  3. ★ここにTEXファイル名★ の 2 箇所を上の表の「中に書く TEX ファイル名」に書き換え
  4. 「ファイル → 名前を付けて保存」で上の表のファイル名で保存
<Atlas><Texture filename="★ここにTEXファイル名★" /><Elements><Element name="★ここにTEXファイル名★" u1="0" u2="1" v1="0" v2="1" /></Elements></Atlas>
保存時の注意

メモ帳の「名前を付けて保存」でファイルの種類を「すべてのファイル (*.*)」に変更してから保存してください。そうしないと .txt が余計に付きます。

例: modicon.xml の場合

テンプレートの ★ここにTEXファイル名★ を modicon.tex に書き換え、ファイル名を modicon.xml として保存:

<Atlas><Texture filename="modicon.tex" /><Elements><Element name="modicon.tex" u1="0" u2="1" v1="0" v2="1" /></Elements></Atlas>

この手順を 7 回繰り返して、すべての XML ファイルを作成してください。

Step 10

ゲーム内でテストする

MOD フォルダをゲームに配置する

  1. 完成した my_character フォルダを DST の mods フォルダにコピー（Steam → Don't Starve Together を右クリック → 管理 → ローカルファイルを閲覧 → mods フォルダ）
  2. mods フォルダ内の dedicated_server_mods_setup.lua は触らない

MOD を有効にしてテストする

  1. DST を起動する
  2. メインメニューから「ゲームをホスト」を選択
  3. 「新しい世界を作成する」をクリック
  4. 上部の「MOD」タブを開く
  5. 自分の作った MOD のチェックボックスをチェックして有効にする
  6. 「ワールドを作成」をクリック
  7. キャラクター選択画面で自分の MOD キャラクターを選択
  8. ゲームが開始され、キャラクターが正常に動けることを確認できれば成功
エラーが出たら: ログファイルの確認

ログファイルをメモ帳で開き、末尾付近にエラーメッセージが出ていないか確認してください。

ログファイルの場所

C:\Users\<ユーザー名>\Documents\Klei\DoNotStarveTogether\client_log.txt

エクスプローラーのアドレスバーに %USERPROFILE%\Documents\Klei\DoNotStarveTogether と入力すると直接開けます。

Troubleshooting

よくあるトラブルと解決法

ゲームが起動しない / クラッシュする
症状原因解決法
MOD を有効にするとクラッシュAssets に登録したファイルが存在しない全ファイルの存在を確認
キャラ選択でクラッシュ画像サイズが 2 の累乗でない画像を 64, 128, 256, 512 にリサイズ
ゲーム開始時にクラッシュprefab ファイルの文法エラーclient_log.txt を確認して修正
MOD が一覧に表示されない

modinfo.lua が MOD フォルダの直下にあるか確認してください。文法エラー（カンマ抜け、引用符の閉じ忘れ）がないか確認してください。dst_compatible = true が設定されているかも確認してください。

キャラクターが透明 / 真っ黒になる

anim/my_character.zip が正しくビルドされているか確認してください。prefab の AnimState:SetBuild() と ZIP のファイル名（拡張子なし）が一致しているか、AnimState:SetBank("wilson") が正しく設定されているかも確認してください。

autocompiler が動かない

exported フォルダに Spriter プロジェクト（.scml + PNG フォルダ）を配置しているか確認してください。管理者権限で実行してみてください。パスに日本語が含まれていないかも確認してください。

パスに日本語が含まれている

Windows のユーザー名が日本語の場合、一部のツールが正常に動作しないことがあります。Steam のインストール先を D:\Steam\ など日本語を含まないパスに変更するか、ゲームのインストール先だけを移動してください（Steam → ゲームを右クリック → プロパティ → インストールフォルダの移動）。

よくあるエラーメッセージと対処法

PANIC: Error loading module '...'

指定された Lua ファイルが見つからない、または文法エラーがある。ファイルが scripts/ フォルダ内の正しい場所にあるか、ファイル名の大文字・小文字が一致しているか確認してください。

Could not find anim [...]

アニメーション ZIP が見つからない。anim/ フォルダに ZIP があるか、modmain.lua の Assets のパスと一致しているか確認してください。

attempt to index a nil value

存在しない変数やテーブルにアクセスしようとしている。エラーメッセージのファイル名と行番号を確認し、コンポーネントの追加順序が正しいか確認してください。

デバッグ用 print 文

Lua コード内に print() を挿入すると、ログファイルに出力されます。

print("=== my_character: master_postinit called ===")
print("Health:", inst.components.health.maxhealth)

ログファイルで === を検索すれば、自分の出力を素早く見つけられます。

DST のゲームファイルを参照する

DST のスクリプトは以下の ZIP にバンドルされています。

C:\Program Files (x86)\Steam\steamapps\common\Don't Starve Together\data\databundles\scripts.zip

scripts\
├── prefabs/ ← キャラクター・アイテムの定義
├── components/ ← コンポーネント（機能の部品）
├── speech_wilson.lua ← ウィルソンのセリフ
└── speech_willow.lua ← ウィロウのセリフ

C:\Program Files (x86)\Steam\steamapps\common\Don't Starve Together\data\anim\
