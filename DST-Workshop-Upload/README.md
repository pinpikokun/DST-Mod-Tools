## Steam ワークショップとは

Steam ワークショップは、ユーザーが作成したゲームの MOD を共有・配布できる Steam の公式プラットフォームです。

ワークショップに MOD を公開すると、以下のメリットがあります。

- 他のプレイヤーが「サブスクライブ」ボタンひとつで MOD をインストールできる
- MOD のアップデートが自動で全ユーザーに配信される
- コメント欄でフィードバックやバグ報告を受け取れる
- 評価やサブスクライブ数で、どれだけ遊んでもらえているかがわかる

> **ポイント**: 心配いりません。ワークショップへの公開は無料で、いつでも非公開に戻したり削除したりできます。気軽に公開してみましょう。

## 公開前の準備

ワークショップにアップロードする前に、以下の項目を確認してください。

### modinfo.lua の確認

MOD のルートフォルダにある `modinfo.lua` の以下の項目が正しく設定されているか確認します。

```lua
-- ★書き換え: ワークショップに表示される情報
name = "My Character MOD"           -- MOD の名前
description = "A custom character."  -- MOD の説明
author = "YourName"                  -- 作者名
version = "1.0.0"                    -- バージョン番号
```

> **重要**: `version` はアップロードのたびに数字を上げる必要があります（例: `"1.0.0"` → `"1.0.1"`）。同じバージョン番号のままアップロードすると、エラーになることがあります。

### MOD アイコンの確認

MOD のルートフォルダに以下の 2 つのファイルがあることを確認してください。

| ファイル | 説明 |
|---|---|
| `modicon.tex` | MOD アイコンのテクスチャファイル |
| `modicon.xml` | MOD アイコンのアトラスファイル |

これらはゲーム内の MOD 一覧に表示されるアイコンです。PNG から TEX への変換方法は [DST-TextureConverter ガイド](../DST-TextureConverter/README.md) を参照してください。

### プレビュー画像の用意

ワークショップページに表示されるサムネイル画像を用意します。

- **形式**: JPG または PNG（TEX ファイルではありません）
- **推奨サイズ**: 512 x 512 以上
- **ファイルの場所**: MOD フォルダの中でなくても構いません（アップロード時に指定します）

> **なぜ TEX ではないのか？**: ワークショップのプレビュー画像は Steam のウェブページに表示されるものなので、通常の画像形式（JPG / PNG）を使います。ゲーム内で使う TEX ファイルとは別のものです。

> **初心者向けTip:** まだプレビュー画像がなくても大丈夫です。MOD アイコンと同じ画像の PNG 版を使っても問題ありません。あとから差し替えることもできます。

### ゲーム内テスト

公開する前に、必ずゲーム内で MOD が正しく動作するか確認してください。

1. DST を起動する
2. MOD を有効化してゲームを開始する
3. キャラクターが正しく表示されるか確認する
4. アイテムやクラフトが動作するか確認する
5. マルチプレイでホストとクライアント両方で問題がないか確認する

### 公開前チェックリスト

| 項目 | 確認内容 |
|---|---|
| `modinfo.lua` | name、description、author、version が正しい |
| MOD アイコン | `modicon.tex` と `modicon.xml` が存在する |
| プレビュー画像 | JPG または PNG ファイルを用意した |
| ゲーム内テスト | シングルプレイで動作確認した |
| マルチプレイテスト | ホスト・クライアント両方で動作確認した |
| 不要ファイル | `exported/` フォルダなどの不要ファイルを削除した |

> **ポイント**: `exported/` フォルダは Spriter がアニメーションをエクスポートする際に作成される作業用フォルダです。MOD の動作には不要なので、アップロード前に削除しておくとファイルサイズを節約できます。

## Mod Uploader の起動

MOD をワークショップにアップロードするには、Steam に付属の **Mod Uploader** ツールを使います。

### 起動手順

1. Steam を開く
2. ライブラリを選択する
3. 検索バーの横にある絞り込みで「ツール」にチェックを入れる
4. 検索バーに `Don't Starve Mod Tools` と入力する
5. **Don't Starve Mod Tools** をクリックして起動する
6. 表示されたメニューから **Mod Uploader** を選択する

> **ヒント**: Don't Starve Mod Tools がインストールされていない場合は、上記の手順で見つけてインストールしてください（無料です）。インストール方法の詳細は [メインガイド](../README.md) を参照してください。

## はじめてのアップロード

Mod Uploader を起動すると、アップロード画面が表示されます。以下の各項目を設定してください。

### 各フィールドの説明

| フィールド | 説明 | 設定内容 |
|---|---|---|
| **Update Data folder** | MOD のファイルが入っているフォルダ | MOD のルートフォルダを指定（例: `mods/my_character/`） |
| **Preview Image** | ワークショップページのサムネイル画像 | 用意した JPG / PNG ファイルを指定 |
| **Workshop Name** | ワークショップに表示される MOD の名前 | `modinfo.lua` の `name` と同じでも、別の名前でも OK |
| **Description** | ワークショップページに表示される説明文 | MOD の機能や特徴を記載 |
| **Tags** | MOD のカテゴリタグ | 該当するものを選択（下記参照） |

### タグの選び方

アップロード画面でタグを選択すると、ワークショップでの検索性が向上します。

| タグ | 用途 |
|---|---|
| **character** | キャラクター MOD |
| **item** | アイテムを追加する MOD |
| **creature** | 生物を追加・変更する MOD |
| **tweak** | ゲームバランスを調整する MOD |
| **scenario** | シナリオ・ゲームモードを追加する MOD |
| **server_admin** | サーバー管理者向けの MOD |

> **ポイント**: キャラクター MOD なら `character` タグを選択してください。複数のタグを選ぶこともできます。

### アップロードの実行

1. すべてのフィールドを設定する
2. **Upload** ボタンをクリックする
3. アップロードが完了するまで待つ
4. 「Success」と表示されたら完了

### アップロード後の確認

アップロードが成功したら、ワークショップページを確認しましょう。

1. Steam を開く
2. DST のワークショップページに移動する
3. 自分の MOD が表示されていることを確認する

> **初心者向けTip:** アップロード直後は「非公開（Hidden）」状態になっている場合があります。ワークショップページの「公開設定」から「公開（Public）」に変更すると、全ユーザーに表示されます。

## MOD の更新（アップデート）

MOD を修正・改善した場合は、ワークショップにアップロード済みの MOD を更新できます。

### 更新手順

1. `modinfo.lua` の `version` を上げる（例: `"1.0.0"` → `"1.0.1"`）
2. Mod Uploader を起動する
3. 既存の MOD を選択する
4. **Update Data** にチェックを入れる
5. MOD フォルダを指定する
6. **Upload** をクリックする

### チェックボックスの使い分け

| チェックボックス | 用途 | 推奨 |
|---|---|---|
| **Update Data** | MOD のファイル（スクリプト、画像等）を更新 | 更新時は毎回チェック |
| **Update Details** | ワークショップの名前・説明・タグを更新 | 名前や説明を変えたい時のみチェック |
| **Update Preview** | プレビュー画像を更新 | サムネイルを変えたい時のみチェック |

> **重要**: **Update Details** はワークショップページの説明文をアップロード画面の内容で上書きします。ワークショップページ上で直接編集した説明がある場合、チェックを入れるとその内容が消えてしまいます。通常は **Update Data** のみチェックすれば十分です。

> **重要**: `version` 番号を上げずにアップロードすると、エラーになることがあります。`modinfo.lua` の `version` を必ず変更してください。

## よくあるエラーと対処法

| 症状 | 原因 | 解決法 |
|---|---|---|
| EResult 9 エラー | 直前のアップロードと同じプレビュー画像を使った | プレビュー画像を別のファイルに変更する（1 ピクセルでも変えれば OK） |
| version 番号エラー | `modinfo.lua` の version を更新していない | version を上げてから再アップロード |
| アップロードが失敗する | MOD のファイルサイズが 100MB を超えている | 不要なファイル（`exported/` フォルダ等）を削除してサイズを減らす |
| Mod Uploader が起動しない | Don't Starve Mod Tools がインストールされていない | Steam ライブラリの「ツール」からインストール |
| アップロード後に MOD が見つからない | 公開設定が「非公開」になっている | ワークショップページで「公開」に変更 |

### ログファイルの確認

アップロードに失敗した場合、ログファイルにエラーの詳細が記録されています。

```
C:\Users\<ユーザー名>\AppData\Local\Don't Starve Mod Uploader\ModUploader.log
```

> **ヒント**: `AppData` フォルダは通常は隠しフォルダです。エクスプローラーの「表示」メニューから「隠しファイル」にチェックを入れると表示されます。

## Tips・参考情報

### 英語の name と description で見つけてもらいやすく

`modinfo.lua` の `name` と `description` を英語で書くと、海外のプレイヤーにも見つけてもらいやすくなります。日本語の説明を添えたい場合は、英語の後に日本語を併記するとよいでしょう。

```lua
name = "My Character MOD"
description = "A custom character with special abilities.\n特殊能力を持つカスタムキャラクター。"
```

### ワークショップ説明文の書式

ワークショップページの説明文では、Steam 独自のリッチテキスト記法が使えます。

| 記法 | 表示 |
|---|---|
| `[h1]見出し[/h1]` | 大見出し |
| `[h2]小見出し[/h2]` | 小見出し |
| `[b]太字[/b]` | **太字** |
| `[i]斜体[/i]` | *斜体* |
| `[list][*]項目1[*]項目2[/list]` | 箇条書きリスト |
| `[url=https://...]リンク[/url]` | リンク |

> **ポイント**: この記法はワークショップページの「説明を編集」で直接入力します。`modinfo.lua` の `description` には使えません。

### 不要ファイルの除外

アップロード前に、以下のようなファイルやフォルダが MOD に含まれていないか確認してください。

| ファイル・フォルダ | 説明 |
|---|---|
| `exported/` | Spriter のエクスポート作業フォルダ |
| `.psd` / `.xcf` | 画像編集ソフトのプロジェクトファイル |
| `.scml` | Spriter のプロジェクトファイル（anim/ 内の .zip があれば不要） |
| `README.md` | 開発用のドキュメント |

> **なぜ不要ファイルを除外するのか？**: ワークショップにはファイルサイズの上限（100MB）があります。また、サブスクライブしたユーザーのダウンロード時間を短くするためにも、不要なファイルは除外しておきましょう。
