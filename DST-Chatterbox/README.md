## 概要

Chatterbox TTS（テキスト音声合成）エンジンを使い、任意のキャラクターの声で新しいセリフを生成するツールです。

---

## 仕組み

Chatterbox TTSは「参照音声の声質」と「テキスト（セリフ）」を組み合わせて新しい音声を生成します。

```
参照音声（wav/）  ─┐
                   ├──→  Chatterbox TTS  ──→  生成音声（output/）
入力テキスト       ─┘
```

重要なポイント：

- **参照音声**が「どんな声で喋るか」を決める（声質・雰囲気）
- **テキスト**が「何を喋るか」を決める（セリフ内容）
- 生成エンジンは確率的に動作するため、**同じ設定でも毎回異なる結果**が出る

---

## フォルダ構成

```
プロジェクトフォルダ/
├── 01_setup.bat          # 初回セットアップ用（仮想環境構築・ライブラリインストール）
├── 02_generate.bat       # 音声生成の実行
├── generate_voice.py     # メインスクリプト（テキスト・参照ファイル・パラメータをここで設定）
├── README.md / .html     # このドキュメント
├── wav/                  # 参照音声ファイルを置くフォルダ
├── output/               # 生成された音声ファイルの出力先（自動）
├── success/              # 気に入った音声を手動で保存するフォルダ
└── venv/                 # Python仮想環境（自動生成・直接編集しない）
```

---

## クイックスタート

1. `01_setup.bat` をダブルクリック（初回1回だけ、5〜10分）
2. `wav/` フォルダに参照音声ファイルを配置する
3. `generate_voice.py` の `LINES` セクションで生成したいセリフを設定する
4. `02_generate.bat` をダブルクリックして音声を生成
5. `output/` フォルダを確認し、気に入ったファイルを `success/` に移動

| 項目 | 要件 |

## セットアップ手順

### ステップ1：Pythonのインストール

[Python 3.11.9](https://www.python.org/downloads/release/python-3119/) をダウンロードしてインストールしてください。

> **初心者向け注意**：インストーラーの最初の画面で **「Add python.exe to PATH」のチェックボックスを必ずオンにしてください**。これをオフのままにすると、コマンドプロンプトからPythonが認識されず、以降のセットアップが全て失敗します。
>
> また、既にPython 3.12など別のバージョンがインストールされている場合は、パスの優先順位によって意図しないバージョンが使われる場合があります。コマンドプロンプトで `python --version` を実行してバージョンを確認してください。

### ステップ2：初期セットアップ

`01_setup.bat` をダブルクリックして実行します。Pythonの仮想環境を作成し、Chatterbox TTS・PyTorchなどのライブラリをインストールします。

- 所要時間：5〜10分（回線速度による）
- インストール中に赤い文字の警告（WARNING）が出ることがありますが、ERROR でなければ基本的に問題ありません

> **初心者向け注意**：プロジェクトフォルダを日本語や全角文字を含むパスに置かないでください（例：`C:\Users\太郎\デスクトップ\`）。英数字のみのパスでないと、ライブラリのインストールや実行時にエラーが発生する場合があります。

### ステップ3：参照音声ファイルの配置

`wav/` フォルダに参照音声ファイルを配置します。詳しくは次の「参照音声ファイルの準備」を参照してください。

---

## 参照音声ファイルの準備

参照音声はAIが「声の雰囲気」を学習するための素材です。この品質が最終結果のクオリティに直結するため、丁寧に準備してください。

### 参照音声の入手方法

キャラクターのゲーム内音声データを使用します。一般的な入手方法：

- **ゲームファイルから直接抽出**：各ゲームのMODツールやデータ展開ツールを使用
- **ファンサイトの音声アーカイブ**：League of Legendsの場合、公式Wikiに音声ファイルがまとまっている場合がある

### 推奨フォーマット

- **形式**：WAVファイル（.wav）
- **サンプルレート**：16kHz（もしくは44.1kHz/48kHzでも動作する場合がある）
- **チャンネル**：モノラルまたはステレオ

### 品質を上げるためのポイント

| ポイント | 説明 |
|---------|------|
| 1セリフ = 1ファイル | 複数のセリフが連結されたファイルは使わない。1つのセリフだけが入ったWAVを用意する |
| ノイズが少ないものを選ぶ | BGMやSEが混入した音声は品質が低下する原因になる |
| 感情・トーンを合わせる | 叫び声の参照で穏やかなセリフを生成すると不自然になりやすい。生成したいセリフの雰囲気に近い参照音声を使う |
| 長すぎない音声を使う | 1〜5秒程度の短いセリフが最も安定した結果を得やすい |

### ファイル名の付け方

ファイル名は自由ですが、以下の点に注意してください：

- **日本語・全角文字は避ける**（パスの問題でエラーになる場合がある）
- 何のセリフか分かる名前にする（例：`Armed_and_ready.wav`、`laugh_01.wav`）

## 使い方

### 生成するセリフの設定

`generate_voice.py` の `LINES` セクションを編集して、生成したいセリフを設定します。

```python
LINES = {
    "yes_sir": {
        "text": "Yes sir!",          # 生成するセリフのテキスト
        "ref": REF_YESSIR,           # 参照音声ファイル（声の雰囲気の基準）
        "exaggeration": 0.5,         # 感情の強度（0.0〜1.0）
        "cfg": 0.5,                  # 参照音声への忠実度（0.0〜1.0）
    },
}
```

行を追加する手順：

1. ファイル上部で参照音声の定数を定義する（例：`REF_MYVOICE = os.path.join("wav", "my_voice.wav")`）
2. `LINES` セクションに辞書を追加する
3. 不要な行はコメントアウト（行頭に `#`）すると生成対象外になる

> **初心者向け注意**：Pythonの辞書はカンマの位置が重要です。各 `}` の後にカンマ `,` を忘れると文法エラーになります。既存の行をコピペして編集するのが安全です。

### 音声を生成する

`02_generate.bat` をダブルクリックして実行します。

- 初回はモデルファイル（数GB）のダウンロードが発生するため、数分〜数十分の待機が必要です
- 2回目以降はダウンロード不要です

| 処理環境 | 1行あたりの所要時間 |
|---------|-------------------|
| CPUのみ | 30秒〜数分（正常な動作です） |
| GPU搭載 | 数秒（NVIDIA GeForce RTX推奨） |

### 結果を保存する

生成されたWAVファイルは `output/` フォルダに自動保存されます。

**同じ名前のファイルは上書きされます**。気に入った結果は `success/` フォルダに移動して保管してください。

> **ポイント**：同じ設定でも実行するたびに異なる結果が出ます。納得のいく結果が出るまで何度か `02_generate.bat` を実行して、複数の候補から最も良いものを選ぶのがおすすめです。

## 音声パラメータの調整

### exaggeration（感情の強度）

| 値 | 効果 |
|----|------|
| 0.0〜0.4 | 落ち着いた、抑えた表現 |
| 0.5〜0.6 | 標準的な表現 |
| 0.7〜1.0 | 活気のある、大げさな表現 |

セリフの内容に合わせて調整してください。例えば、掛け声や笑い声は高めに、ダメージ時のうめき声は低めにすると自然な結果を得やすくなります。

### cfg（参照音声への忠実度）

| 値 | 効果 |
|----|------|
| 0.0〜0.3 | AIの自由度が高い。元の声とかなり異なる結果になりうる |
| 0.4〜0.6 | バランスが取れた結果。まずはこの範囲で試すのがおすすめ |
| 0.7〜1.0 | 元の音声にかなり近い結果になる。ただし高すぎると不自然になる場合もある |

> **コツ**：まずは exaggeration=0.5、cfg=0.5 から始めて、結果を聞きながら少しずつ調整していくと効率的です。一度に大きく変えるより、0.1ずつ変えて違いを確認するのがおすすめです。

## DST MOD への反映

生成したWAVファイルをそのままDSTのMODに組み込むことはできません。**FMOD Designer** というツールを使って、WAVファイルをDSTが読み込める形式（`.fev` + `.fsb`）に変換する必要があります。

FMOD Designerの使い方については、専用のFMOD Designerプロジェクトフォルダ内のREADMEを参照してください。

## よくある質問

### 別のキャラクターのMODを作りたい

このプロジェクトをフォルダごとコピーして、以下を差し替えてください：

1. `wav/` フォルダの参照音声ファイルを対象キャラのものに入れ替える
2. `generate_voice.py` のファイル上部で参照ファイルの定数を新しいファイル名に合わせる
3. `LINES` セクションのテキスト・参照・パラメータを書き換える

スクリプトのファイル名（`generate_voice.py`）を変更する場合は、`02_generate.bat` 内のファイル名も合わせて変更してください。

### 英語以外のテキストは生成できる？

Chatterbox TTSは主に英語に最適化されています。他の言語では品質が低下する可能性がありますが、試す価値はあります。

### GPUを使いたい

`01_setup.bat` のPyTorchインストール行を、GPU対応版に変更する必要があります。詳しくは [PyTorchの公式サイト](https://pytorch.org/get-started/locally/) で、お使いのGPUに合ったインストールコマンドを確認してください。

また、 `generate_voice.py` の `device="cpu"` を `device="cuda"` に変更してください。

## トラブルシューティング

### Pythonが見つからないというエラーが出た場合

Python 3.11がインストールされていないか、PATHに追加されていません。[Python 3.11.9](https://www.python.org/downloads/release/python-3119/) をインストールしてください。インストール時に **「Add python.exe to PATH」にチェック**を入れてください。

その後、`venv/` フォルダを削除して `01_setup.bat` を再実行してください。

### torchやライブラリ関連のエラーが発生した場合

仮想環境が破損している可能性があります。`venv/` フォルダを完全に削除し、`01_setup.bat` を再実行してください。

### 「SyntaxError」が表示された場合

`generate_voice.py` の編集でPythonの文法エラーが発生しています。よくある原因：

- カンマ `,` の付け忘れ
- ダブルクォーテーション `"` の閉じ忘れ
- インデント（空白）の数がずれている

エラーメッセージに表示される行番号付近を確認してください。

### 「Reference audio not found」というエラーが出た場合

`wav/` フォルダに指定されたファイルが見つかりません。以下を確認してください：

- ファイル名のスペルが `generate_voice.py` 上部の定数定義と完全に一致しているか
- ファイルが `wav/` フォルダ直下に置かれているか（サブフォルダに入っていないか）

### 生成処理が遅い場合

CPUのみでの生成は正常な動作です。高速化したい場合は NVIDIA GeForce RTX などのGPU導入を検討してください。

### モデルのダウンロードが失敗した場合

ダウンロード用キャッシュが破損している場合があります。`C:\Users\[ユーザー名]\.cache` フォルダを削除して、`02_generate.bat` を再実行してください。
